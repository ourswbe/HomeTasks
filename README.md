# HomeTasks — система учёта домашних заданий

Веб-приложение для курсовой работы по теме домашних заданий.

## Возможности

- Регистрация и вход с выбором роли: **учитель** или **ученик**.
- Учитель добавляет задания с предметом, дедлайном и типом:
  - обычное задание (текст/ссылка/фото);
  - тестовое задание (**multiple choice**).
- Ученик видит список заданий, фильтрует по предмету и отправляет решение:
  - текст;
  - ссылка на файл/документ;
  - ссылка на фото;
  - выбранный вариант (для теста).
- Учитель видит отправки учеников и может:
  - принять как выполненное;
  - вернуть на доработку;
  - выставить оценку и комментарий.
- Отображаются дедлайны и статусы: на проверке / выполнено.
- Ученик видит оценку и комментарий учителя по отправке.
- Данные сохраняются в базе данных SQLite.

## Технологии

- Python 3.11
- Flask
- Flask-SQLAlchemy
- SQLite
- Docker / Docker Compose

## Запуск локально

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

Приложение будет доступно на: http://localhost:5000

## Запуск через Docker

```bash
docker compose up --build
```

После запуска откройте: http://localhost:5000

## База данных

База создаётся автоматически при первом запуске (`hometasks.db`).

## Основные экраны

- `/register` — регистрация (с выбором роли).
- `/login` — авторизация.
- `/tasks` — список заданий, фильтр по предмету, отправка решений и проверка.
- `/tasks/add` — добавление задания (только для учителя).

## Важно после обновления модели БД

чтобы `db.create_all()` создал таблицы/поля с актуальной структурой.

## Почему GitHub пишет "This branch has conflicts"

Это не ошибка приложения. Это конфликт Git между вашей веткой PR и целевой веткой (обычно `main`):

- в обоих местах изменены одни и те же файлы (`README.md`, `app.py`, `templates/tasks.html` и т.д.);
- GitHub не может автоматически выбрать «чью» версию оставить.

### Как исправить

1. Заберите актуальную целевую ветку.
2. Влейте её в свою ветку PR (или сделайте rebase).
3. Вручную решите конфликтные блоки `<<<<<<<`, `=======`, `>>>>>>>`.
4. Закоммитьте и отправьте изменения.

Пример через merge:

```bash
git checkout <your-pr-branch>
git fetch origin
git merge origin/main
# решить конфликты
git add .
git commit -m "Resolve merge conflicts with main"
git push
```

Пример через rebase:

```bash
git checkout <your-pr-branch>
git fetch origin
git rebase origin/main
# решить конфликты
# затем для каждого шага:
git add .
git rebase --continue
# в конце:
git push --force-with-lease
```
