# HomeTasks — система учёта домашних заданий

Веб-приложение для курсовой работы по теме домашних заданий.

## Возможности

- Регистрация и вход с выбором роли: **учитель** или **ученик**.
- При регистрации указывается **класс** (например, 8A, 9B).
- Учитель может открыть страницу **"Мой класс"** и увидеть учеников своего класса.
- Учитель добавляет:
  - обычное задание (текст/ссылка/фото),
  - тест (multiple choice) с **количеством вопросов**.
- Для теста учитель сначала задаёт число вопросов, затем формирует варианты ответа к каждому вопросу.
- Ученик может выбрать **ровно один вариант по каждому вопросу**.
- После истечения дедлайна отправка блокируется (сдать тест/задание уже нельзя).
- Есть форма **сброса пароля** по email.
- Учитель проверяет решения, выставляет оценку и комментарий.

## Технологии

- Python 3.11
- Flask + Flask-SQLAlchemy
- SQLite (по умолчанию) **или PostgreSQL**
- Docker / Docker Compose

## База данных

По умолчанию используется SQLite (`sqlite:///hometasks.db`).

Чтобы использовать PostgreSQL, задайте переменную `DATABASE_URL`:

```bash
export DATABASE_URL="postgresql+psycopg2://hometasks:hometasks@localhost:5432/hometasks"
```

Приложение читает:
- `DATABASE_URL`
- `SECRET_KEY`

## Запуск локально (SQLite)

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

## Запуск с PostgreSQL через Docker

```bash
docker compose up --build
```

Будет поднято 2 сервиса:
- `db` (PostgreSQL 16)
- `web` (Flask), доступен на http://localhost:5000

## Важно про старые SQLite базы

Если ранее база была создана старой версией приложения, при старте выполняется
авто-добавление недостающих столбцов (чтобы избежать ошибок вида
`sqlite3.OperationalError: no such column: ...`).

## Как уменьшить конфликты на GitHub

1. Перед началом работы обновляйте ветку:
   ```bash
   git checkout <branch>
   git fetch origin
   git rebase origin/main
   ```
2. Делайте маленькие PR (меньше файлов в одном PR = меньше конфликтов).
3. Не редактируйте одновременно один и тот же большой файл несколькими ветками.
4. Перед пушем ещё раз синхронизируйтесь с `main`.
5. Перед коммитом проверьте, что не осталось conflict markers:
   ```bash
   rg "^(<<<<<<<|=======|>>>>>>>)" .
   ```

## Основные экраны

- `/register` — регистрация.
- `/login` — авторизация.
- `/reset-password` — сброс пароля.
- `/tasks` — список заданий, отправка решений, проверка.
- `/tasks/add` — добавление задания (только для учителя).
- `/classroom` — список учеников класса учителя.
